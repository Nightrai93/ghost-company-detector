# 🛡️ GhostCompanyDetector — Anti-Synthetic Corporate KYC API

An enterprise-grade, high-performance B2B fraud prevention API designed to detect **AI-generated "ghost" companies** and synthetic business identities during SaaS registration and onboarding.

Built with **FastAPI**, **HTTPX**, and **Supabase**, featuring advanced network-level security against SSRF and DNS Rebinding attacks.

---

## 💡 The Problem in 2026
With the explosion of LLMs and autonomous AI agents, fraudsters can spin up highly convincing fake businesses in seconds. They generate flawless landing pages, faked "About Us" stories, and placeholder social profiles to bypass automated onboarding checks. Once inside, they exploit cloud infrastructure, commit subscription fraud, or launder resources.

Traditional KYC tools check if a domain exists, but they cannot tell if a business was built by a human or generated by a prompt.

---

## ✨ Core Features
- **AI Marketing Footprint Analysis:** Scans webpage copy for repetitive LLM linguistic patterns and cliches used by ChatGPT/Claude.
- **Social Placeholder & Dead Link Detection:** Heuristically scans DOM elements to spot unconfigured social media anchors (e.g., `href="#"` on LinkedIn/Twitter icons), a major indicator of mass-generated synthetic sites.
- **Corporate Trust Scoring:** Automatically aggregates signals into a single `corporate_trust_score` (1-100) and assigns a clear `risk_level` (`LOW`, `MEDIUM`, `CRITICAL`).
- **Military-Grade SSRF Protection:** Implements a custom `PinnedIPTransport` layer that resolves DNS before fetching and enforces strict IP pinning, neutralizing Time-of-Check to Time-of-Use (TOCTOU) and DNS Rebinding exploits.
- **Sub-Millisecond Atomic Cache:** Powered by Supabase PostgreSQL upserts to ensure lightning-fast responses and zero overhead for repeated URLs.

## 🔒 Security Architecture

Since this API fetches arbitrary user-submitted URLs, it has been hardened against common infrastructure vulnerabilities:

1. **DNS Pre-Resolution:** The server resolves the target hostname to an IP address before establishing a connection.
2. **IP Blacklisting:** Private, loopback, and local IP ranges (e.g., `10.0.0.0/8`, `192.168.0.0/16`, `127.0.0.1`) are explicitly blocked to prevent **Server-Side Request Forgery (SSRF)**.
3. **IP Pinning:** Forces the HTTP client to connect *only* to the pre-verified IP address, entirely eliminating **DNS Rebinding** hacks.
4. **Memory Guard:** Response content is processed in strict 8KB streaming chunks with a 3MB hard cutoff, protecting the application layer from memory-exhaustion attacks (Zip bombs/Giant pages).
